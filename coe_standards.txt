1. Overview
This document provides a comprehensive template and guidelines for creating Python projects that adhere to
GenWorx Center of Excellence (COE) standards. It covers both traditional Python applications and AI-powered
projects with modular, scalable architectures.
Supported Project Types
Standard Python Projects: REST APIs, CLI tools, data processing pipelines.
gRPC Services: High-performance internal microservices.
Core Philosophy
Modularity: Clear separation of concerns with well-defined module boundaries.
Scalability: Architecture supports horizontal and vertical scaling.
Maintainability: Code is clean, documented, and follows consistent patterns.
Security: Security-first approach with secrets management and vulnerability scanning.
Observability: Built-in logging, metrics, and tracing capabilities.
Async-First: All I/O-bound operations must use async/await.

6. Security Guidelines
6.1 Secrets
NEVER commit secrets.
Use pydantic-settings to read from environment.
6.2 API Security
Auth: OAuth2 / OIDC with JWT.
Validation: Strict Pydantic models for all inputs.
Rate Limiting: Use slowapi to prevent abuse.
6.3 Docker Security
Run as non-root user (USER appuser).
Use multi-stage builds to minimize image size.
Scan images regularly with Trivy.
7. Documentation Standards
7.1 Code
Docstrings: Google Style docstrings for all public APIs.
Type Hints: Mandatory for self-documentation.
7.2 API
OpenAPI: Auto-generated by FastAPI.
Descriptions: Add summary and description to all routes.
8. Troubleshooting
Common Issues
Module Not Found: Ensure uv pip install -e . was run.
DB Connection: Check Docker container status (docker ps).
